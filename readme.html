Assign Order to Customer

Version 1.1

Assign Order to Customer - is a tool that allows you to assign orders from Guest or Customer to another customer.

Features

  • Ability to assign guest order to any registered customer
  • Ability to assign any order from one customer to another
  • Ability to assign multiple orders to customer using mass actiion
  • Ability to view a history of assigned guest orders
  • Ability to rollback the assignment to any history point
  • Order Assignment email notification to customer and administrator
  • Easy 2-minutes installation
  • Free support
  • Meets Magento programming practices for most versions and customizations compatibility

Installation

Before you start, we would like to point out that if you are not confident performing the installation yourself, you should let the team of Magpleasure Ltd. do it. People who developed the extension would be the best choice and they usually offer this service for a small fee. See http://www.magpleasure.com/installation-service.html.

Backup your data

Make a Backup copy of the files and database for your Magento store.

Disable Compilation

If you have compilation enabled, please disable it. Installing extension with enabled compilation will inevitably lead to problems in the front end or the backend. This can be corrected, but would require an expert help. So to avoid it, temporary disable the compilation.

Enable all Types of Cache

If you disable the cache, then turn it on and refresh it. It is best to install extensions to the store, when no one except you have access to it during that time, because installation process can begin before all extensions files are loaded. This may lead to some problems.

Download and unzip the package

Download and unzip the extension package from the website where you bought it from. On our site you can get this package from My Downloadable Products in the customer area.

Once the package is unpacked, go into the unpacked directory

Upload files to the server. Upload app, js and skin folders to the server using an FTP client. If you are using a modified theme or a custom template, you will need to make an additional copy of the directory app/design/frontend/default/default to app/design/frontend/<YOUR_PACKAGE>/<YOUR_THEME>.

After that copy skin/frontend/default/default to skin/frontend/<YOUR_PACKAGE>/<YOUR_THEME>.

Depending on the extension function, some folders may not exist. In this case, they do not need to be copied.

Additional installation steps

Some extensions require you to make additional modifications to the source files of your store. Find the Installation section in the readme.html file, which comes with each extension. If any, here you would find any additional modifications.

Refresh Store Cache

Unless you refresh the store cache, the extension will not be installed properly. Go to System → Cache Management, and click Flush Cache Storage. The cache is cleared and the extension will be installed to the store after refreshing of any page.

That's done

The extension is installed to your store and you can start configuring it if you need. If you were using compilation, you can enable it now.

Configuration

After installation you have the ability to customize customer email notifications. To do this, go to System->Configuration->Assign Order to Customer. Here you can enable the notification of the assigned order, select the Email Template and choose to receive a copy of the email that is sent to customer.

For store administrators who are allowed to assign guest orders to customers, you need to enable the permission Assign to Customer under Sales->Order role permissions group. 

Troubleshooting

After the extension installation the store gives an error, or blank page, or suggests to start Magento installation procedure.
Change the owner of the extracted extension files to the web server user and set 775 permissions on them. Clear the store cache and try again.
There is no MagPleasure extensions under my configuration section, or having the extension tab clicked I get a blank page, or Access Denied error.
Clear the store cache, browser cookies, logout and login again.
I've set up everything correctly, inserted the HTML code but there is nothing on that page.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
My configuration changes do not appear on the store.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
